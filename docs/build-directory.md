# Build directory reference

Everything under `build/` is generated by CMake and the compiler. You never edit these files by hand. Delete the whole directory anytime with `rm -rf build` and recreate it with:

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
```

This document describes **every file** produced by a full configure + build of qmog.cpp (83 files, ~3.7 MB). Generated with CMake 4.3.0 using the **Unix Makefiles** generator on macOS arm64.

---

## Top level (`build/`)

| File | Purpose |
|------|---------|
| `CMakeCache.txt` | Persistent cache of all CMake variables from the last configure run: compiler path (`/usr/bin/c++`), build type (`Release`), flags, OpenMP paths, generator type, etc. Speeds up re-runs of `cmake` and lets you inspect or override settings without re-detecting everything. |
| `Makefile` | The main build recipe. Running `cmake --build build` (or `make` from inside `build/`) executes rules defined here. Includes compile and link commands for `qmog-cli` and `test_exec`. |
| `cmake_install.cmake` | Install script generated for `cmake --install build`. Defines where binaries would be copied (default prefix `/usr/local`). This project does not define install targets, so the script is mostly boilerplate. |
| `qmog-cli` | **The CLI executable.** Linked from 9 object files. This is what you run: `./build/qmog-cli model.mog "Hello"`. |
| `test_exec` | **The test runner executable.** Linked from 14 object files (shared sources + test/qwen sources). Run with `./build/test_exec`. |

---

## CMake internal (`build/CMakeFiles/`)

These files are CMake's own bookkeeping. Make reads them indirectly via the top-level `Makefile`.

| File | Purpose |
|------|---------|
| `CMakeConfigureLog.yaml` | Verbose log of the configure step: every program CMake searched for (`uname`, `c++`, etc.), compiler detection results, and probe outcomes. Useful when configure fails; otherwise ignorable. |
| `CMakeDirectoryInformation.cmake` | Sets path conventions for this build directory: source root, binary root, and regex rules for parsing `#include` lines in dependency scanning. |
| `InstallScripts.json` | JSON manifest listing install scripts CMake knows about. Points at `cmake_install.cmake`. Used internally by CMake's install machinery. |
| `Makefile.cmake` | Dependency list for the top-level `Makefile`. Records which inputs (CMakeLists.txt, CMakeCache.txt, CMake system modules) would trigger a reconfigure if changed. The long list of `/opt/homebrew/share/cmake/Modules/...` paths are **references**, not copies. |
| `Makefile2` | Secondary makefile defining the target graph: which targets exist (`qmog-cli`, `test_exec`, `edit_cache`, `rebuild_cache`), their dependencies, and how they relate. The top-level `Makefile` includes this. |
| `TargetDirectories.txt` | Simple list of per-target build directories. One line per target: `qmog-cli.dir`, `test_exec.dir`, plus utility targets `edit_cache.dir` and `rebuild_cache.dir`. |
| `VerifyGlobs.cmake` | Tracks `file(GLOB_RECURSE ...)` results from `CMakeLists.txt`. Compares the current glob of `src/*.cpp` and `test/qwen/*.cpp` against a saved snapshot. If you add or remove a source file, this triggers reconfigure. Exists because of `CONFIGURE_DEPENDS` on the globs. |
| `cmake.check_cache` | Stub file whose timestamp CMake checks to detect manual edits to `CMakeCache.txt`. If the cache file is newer than this stub, CMake re-runs configure. |
| `cmake.verify_globs` | Stub file touched when `VerifyGlobs.cmake` detects a glob mismatch. Its timestamp tells CMake to reconfigure on the next build. |
| `progress.marks` | Single number (`24`) — total build steps across all targets. Used by Make to calculate and display `[ 41%]` style progress during `cmake --build`. |

### Empty directories (no files, but present)

| Directory | Purpose |
|-----------|---------|
| `CMakeScratch/` | Temporary workspace for CMake try-compile probes during configure. Empty after configure completes. |
| `pkgRedirects/` | Placeholder for CMake package redirect files (used when overriding find_package locations). Unused in this project. |

## Compiler detection (`build/CMakeFiles/4.3.0/`)

Created once during the first `cmake -S . -B build`. The `4.3.0` folder name matches the CMake version. Not updated on incremental builds.

| File | Purpose |
|------|---------|
| `CMakeSystem.cmake` | Detected platform: `Darwin` (macOS), processor `arm64`, host and target system names. |
| `CMakeCXXCompiler.cmake` | Detected C++ compiler: AppleClang 21.0.0, path `/usr/bin/c++`, supported `-std=` levels (c++17, c++20, etc.), sysroot path, architecture. All the `CMAKE_CXX_*` variables used later. |
| `CMakeDetermineCompilerABI_CXX.bin` | Binary output from a test compile that determines the compiler's Application Binary Interface (struct layout, name mangling, etc.). CMake reads metadata from this to set linker and flag variables correctly. |
| `CompilerIdCXX/CMakeCXXCompilerId.cpp` | Source of a test program CMake compiles to identify the compiler vendor (GCC, Clang, MSVC, etc.) via preprocessor macros. Standard CMake boilerplate, ~950 lines. |
| `CompilerIdCXX/apple-sdk.cpp` | Additional probe source for detecting the Apple SDK / sysroot during compiler identification on macOS. |
| `CompilerIdCXX/a.out` | Compiled output of the compiler ID probe. Confirms the compiler works and reveals its identity. |
| `CompilerIdCXX/tmp/` | Empty temp directory used during the compiler ID compile step. |

## Target: `qmog-cli` (`build/CMakeFiles/qmog-cli.dir/`)

One set of these files exists per executable target. These 8 files tell Make how to build `qmog-cli`.

| File | Purpose |
|------|---------|
| `build.make` | The actual Make rules for this target: one rule per `.cpp` → `.o` compile, plus the link rule that produces `qmog-cli`. This is where the real compile commands live. |
| `flags.make` | Compiler flags for this target: `-O3 -mcpu=native -Xpreprocessor -fopenmp -I/opt/homebrew/opt/libomp/include -I.../include -std=gnu++17 -arch arm64`. |
| `link.txt` | Exact linker command: compiler path, flags, all `.o` file paths, output `qmog-cli`, and `-lomp`. |
| `DependInfo.cmake` | Maps each source file to its object file and dependency file. Used by CMake's dependency scanner and `make clean`. |
| `depend.make` | Makefile fragment for inter-target dependencies (e.g. "this object depends on that generated header"). Empty for this project — no generated sources. |
| `compiler_depend.make` | Makefile fragment for header dependencies discovered by the compiler (`-MMD`). Starts empty; populated as the compiler reports `#include` changes. |
| `compiler_depend.ts` | Timestamp file tracking when dependency scanning last ran. Make compares this to decide whether to rescan headers. |
| `cmake_clean.cmake` | CMake script listing every file to delete for `make clean`: all `.o`, `.d`, and the `qmog-cli` binary. |
| `progress.make` | Maps build steps to progress numbers (`CMAKE_PROGRESS_1` through `_10`) for the 9 compile steps + 1 link step. |

Same 8 metadata files as `qmog-cli`, but for the test executable. Contents are identical in structure; flags differ slightly (extra include path for `test/qwen`).

| File | Purpose |
|------|---------|

| `DependInfo.cmake` | Source → object mapping for all 14 inputs. |
| `depend.make` | Inter-target dependency fragment. Empty. |
| `compiler_depend.make` | Header dependency fragment. Starts empty. |

| `cmake_clean.cmake` | File list for `make clean` on this target. |

## Summary

| Category | Files | Needed to run? |
|----------|-------|----------------|
| Executables | 2 | **Yes** — these are the whole point |
| Object files (`.o`) | 22 | No — intermediate compile output |
| Dependency files (`.d`) | 22 | No — incremental rebuild bookkeeping |
| CMake/Make metadata | 34 | No — build system internals |
| Compiler probes | 5 | No — one-time configure artifacts |
| **Total** | **83** | **2 matter for running the program** |

Everything except `qmog-cli` and `test_exec` exists so that the next `cmake --build build` only recompiles files that actually changed.
